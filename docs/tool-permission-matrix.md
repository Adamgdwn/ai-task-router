| Tool | Purpose | Allowed Actions | Prohibited Actions | Approval Required | Notes |

| Browser app | Recommend AI task routes from user-entered setup and task details. | Read/write browser-local IndexedDB, generate local route cards, prompt packages, exports, and route logs. | Provider API calls, account connections, credential storage, telemetry, hidden uploads, file indexing, external actions. | User action for exports, saves, resets, and feedback entry. | Current v0.2 surface. |
| `npm run detect:local-models` | Explicit local terminal check for common local model tools. | Read-only checks for Ollama and common local model folders; summary output by default. | Browser state changes, network calls, provider login, file content indexing, hidden telemetry, arbitrary shell execution. | User must run the command manually. | `--details` and `--json` can reveal more local details and should be used intentionally. |
| Tauri desktop shell | Run the existing React app inside a desktop window. | Load bundled app assets, use one main window, apply an explicit CSP, and use an empty default capability list. | Native discovery, filesystem reads, shell/process execution, uploads, telemetry, updater, provider connections, credentials, file indexing, packaging/signing claims. | None beyond normal user launch; no native access prompts are present. | Current D2/D3 surface. No-bundle build passes; `desktop:dev` and the current rebuilt unsigned release executable are blocked by Windows Application Control. |
| Future desktop discovery options command | Let the desktop UI show which allowlisted local AI checks are available. | Return OS/platform and fixed tool options using `desktopDiscoveryOptionsResponseSchema`. | Scanning folders, listing models, reading files, running commands, connecting providers, collecting telemetry. | User must open the desktop discovery flow before this matters in the UI. | Planned D4 command: `get_desktop_discovery_options`. Not implemented yet. |
| Future desktop native local discovery | Let a signed desktop app check selected local AI tools and model folders. | Read-only, user-approved, allowlisted checks with timeouts and summarized results validated by `desktopDiscoveryRequestSchema` and `desktopDiscoveryResponseSchema`. | Silent scanning, startup/background inventory, whole-drive search, file-content reading, full path disclosure, arbitrary shell text, uploads, telemetry, credentials, provider actions, destructive actions. | User must explicitly choose what to check before each discovery run; summary appears before optional details. | Planned D4 command: `run_desktop_discovery`. D3 permits only the documented contract, not implementation. See `docs/2026-07-04-desktop-trust-distribution-plan.md` and `src/domain/schemas.ts`. |
